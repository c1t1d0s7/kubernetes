### 1) 디플로이먼트 정의

#### (1) 디플로이먼트 기본 사양
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: mynapp-deploy
  labels:
    app: mynapp-deploy
spec:
  replicas: 3
  selector:
    matchLabels:
      app: mynapp-deploy
  template:
    metadata:
      labels:
        app: mynapp-deploy
    spec:
      containers:
      - image: c1t1d0s7/myweb:v1
        name: mynapp
        ports:
        - containerPort: 8080
```

#### (2) 디플로이먼트 전략
```yaml
spec:
  strategy:
    type: [RollingUpdate | Recreate]
    rollingupdate:
      maxUnavailable: <정수 | 비율>
      maxSurge:
```
- RollingUpdate: 롤링업데이트 방식(기본값)
    - maxUnavailable: 업데이트 프로세스 중에 사용할 수 없는 최대 파드의 수(기본값: 25%)
        - 정수: 예) 3
        - 비율: 예) 10%
    - maxSurge: 생성할 수 있는 최대 파드의 수(기본값: 25%)
        - 정수: 예) 3
        - 비율: 예) 10%
- Recreate: 재생성(기존 파드는 모두 삭제 되고, 새로운 파드 생성)

#### (3) 컨테이너 최소 대기 시간
```yaml
spec:
  minReadySeconds: <초>
```
- minReadySeconds: 새로 배포된 컨테이너가 준비되기 까지 대기할 시간(기본값: 0초)

### 2) 디플로이먼트 생성
> mynapp-deploy-v1.yml
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: mynapp-deploy
  labels:
    app: mynapp-deploy
spec:
  strategy:
    type: RollingUpdate
    rollingupdate:
      maxUnavailable: 1
      maxSurge: 1
  minReadySeconds: 5
  replicas: 3
  selector:
    matchLabels:
      app: mynapp-deploy
  template:
    metadata:
      labels:
        app: mynapp-deploy
    spec:
      containers:
      - image: c1t1d0s7/myweb:v1
        name: mynapp
        ports:
        - containerPort: 8080
```

```
kubectl create -f mynapp-deploy-v1.yml --record
```

### 3) 디플로이먼트 롤아웃 상태 확인
```
kubectl rollout status deployment mynapp-deploy
```

```
kubectl rollout history deployment mynapp-deploy
```

### 4) 디플로이먼트 롤링업데이트

#### (1) 롤링업데이트 명령 형식
```
kubectl set image deployment <디플로이먼트 이름> <컨테이너 이름>=<새 이미지>
kubectl set image deployment -f <디플로이먼트 파일> <컨테이너 이름>=<새 이미지>
```

#### (2) 버전1 -> 버전2
```
kubectl set image deployement mynapp-deploy mynapp=c1t1d0s7/myweb:v2
```

```
kubectl rollout status deployment mynapp-deploy
```

```
kubectl rollout history deployemt mynapp-deploy
```

#### (5) 버전2 --> 버전3
> mynapp-deploy-v3.yml
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: mynapp-deploy
  labels:
    app: mynapp-deploy
  annotations:
    kubernetes.io/change-cause: My Node App Version 3
spec:
  strategy:
    type: RollingUpdate
    rollingupdate:
      maxUnavailable: 1
      maxSurge: 1
  minReadySeconds: 5
  replicas: 3
  selector:
    matchLabels:
      app: mynapp-deploy
  template:
    metadata:
      labels:
        app: mynapp-deploy
    spec:
      containers:
      - image: c1t1d0s7/myweb:v3
        name: mynapp
        ports:
        - containerPort: 8080
        readinessProbe:
          periodSeconds: 1
          httpGet:
            path: /
            port: 8080
```

!!!어노테이션 change-cause 적용되는지 확인!!!
```
kubectl apply -f mynapp-deploy-v3.yml
```

```
kubectl rollout history deployment mynapp-deploy-v3.yml
```
