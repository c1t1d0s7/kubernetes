## 1. 레플리케이션 컨트롤러 롤링업데이트

### 1) 롤링업데이트를 위한 리소스 준비
> mynapp-rc-v1.yml
```yaml
apiVersion: v1
kind: ReplicationController
metadata:
  name: mynapp-rc-v1
spec:
  replicas: 3
  template:
    metadata:
      labels:
        app: mynapp-rc
    spec:
      containers:
      - name: mynapp
        image: c1t1d0s7/myweb:v1
        ports:
        - containerPort: 8080
```

> mynapp-svc.yml
```yaml
apiVersion: v1
kind: Service
metadata:
  name: mynapp-svc-roll
spec:
  type: LoadBalancer
  ports:
  - port: 80
    targetPort: 8080
  selector:
    app: mynapp-rc
```

```
kubectl create -f mynapp-rc-v1.yml -f mynapp-svc.yml
```

```
watch -n1 -d kubectl get all
```

```
watch -n1 -d curl http://X.X.X.X
```

### 2) 롤링업데이트
```
kubectl rolling-update <현재 컨트롤러> <새 컨트롤러> --image=<새 이미지>
kubectl rolling-update <현재 컨트롤러> <새 컨트롤러> -f <새 오브젝트 파일>
```

```
$ kubectl rolling-update mynapp-rc-v1 mynapp-rc-v2 --image=c1t1d0s7/myweb:v2
```

```
kubectl get pods --show-labels
```

```
kubectl get replicationcontroller -o wide
```
