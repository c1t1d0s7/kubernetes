## 3. 어노테이션

### 1) 어노테이션 소개
어노테이션(Annotation)은 주석이라고도 하며 오브젝트에 메타데이터를 첨부할 수 있다. 레이블도 비슷하게 키/값을 가지고 있지만, 레이블은 레이블 셀렉터를 이용하여 식 별 및 검색을 할 수 있지만, 어노테이션은 어노테이션 셀렉터를 가지고 있지 않다. 즉, 메타데이터를 추가해 추가적인 정보를 제공하기 위함이다.

또한 어노테이션은 쿠버네티스 클러스터나 클라이언트 및 라이브러리가 어노테이션에 지정된 키/값을 참조하여 동작하는 경우도 있다.

어노테이션 사용 예:
- 선언적 구성정보
- 타임 스탬프, 릴리즈 ID, git 브랜치, PR 번호, 이미지 해시, 레지스트리 주소
- 로깅, 모니터링, 분석, 감사 정보
- 디버깅정보
- 다른 요소와의 관련 개채 URL 지정
- 책임자, 관리자 정보

### 2) 어노테이션 추가 및 수정
레이블과 마찬가지로 오브젝트 생성 시 선언할 수도 있고, 오브젝트가 생성된 후에도 추가 및 수정할 수 있다.

기존 mynapp-pod 파드에 devops-team/developer 란 어노테이션을 추가해 보자.
```
$ kubectl annotate pods mynapp-pod devops-team/developer="John Smith"

pod/mynapp-pod annotated
```

어노테이션 확인은 kubectl describe 명령을 사용하여 확인할 수 있다.
```
$ kubectl describe pods mynapp-pod

Name:         mynapp-pod
Namespace:    default
Priority:     0
Node:         kube-node2/192.168.56.22
Start Time:   Mon, 16 Mar 2020 10:59:29 +0000
Labels:       env=dev
Annotations:  devops-team/developer: John Smith
```

역시 kubectl get 명령으로도 확인이 가능하다.
```
$ kubectl get pods mynapp-pod -o yaml

apiVersion: v1
kind: Pod
metadata:
  annotations:
    devops-team/developer: John Smith
  creationTimestamp: "2020-03-16T10:59:29Z"
  labels:
    env: dev
```

YAML 파일로 선언할 때 pods.metadata.annotations 필드에 선언 할 수도 있다.
