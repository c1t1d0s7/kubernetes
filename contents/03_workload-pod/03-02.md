## 2. 레이블 및 셀렉터

### 1) 레이블 소개
레이블(Label)은 쿠버네티스 클러스터의 모든 오브젝트(파드 포함)에 키/값 쌍으로 리소스를 식별하고 속성을 지정하는 데 사용한다. 

오브젝트 개수가 많아진다면 오브젝트를 식별하는데 매우 어려울 수 있으며, 오브젝트의 적절한 레이블을 부여하여 성격을 정의하고 검색을 용이하게 할 수 있다.

레이블은 사용자에게는 중요하지만, 쿠버네티스 클러스터에 직접적인 의미는 없다.

레이블의 예:
- release: stable / canary / A / B 
- environment: dev / qa / production
- tier: frontend / backend / cache / database
- partition: customerA / customerB
- track: daily / weekly
- app: webapp / middleware

레이블 구문:
키는 "접두사(옵션)/이름" 형식으로 사용 가능하며, 영문자, 숫자, 대시(-), 언더스코어(_), 점(.)이 올 수 있습니다. 이름은 최대 63자를 넘지 못한다.

> 참고
> `kubernetes.io/`와 `k8s.io/` 접두사는 쿠버네티스 핵심 구성요소에서 예약됨

### 2) 레이블을 사용한 파드 정의 
> myapp-pod-label.yaml
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp-pod-label
  labels:
    env: dev
    tier: frontend
spec:
  containers:
  - image: ghcr.io/c1t1d0s7/go-myweb
    name: myapp
    ports:
    - containerPort: 8080
      protocol: TCP
```

env와 tier 레이블을 추가하여 myapp-pod-label 파드를 생성한다.

### 3) 파드 생성
파드를 생성하자.
```
$ kubectl create -f myapp-pod-label.yaml

pod/myapp-pod-label created
```

### 4) 파드 레이블 확인
kubectl get pods 명령에 --show-labels 옵션을 사용하면 파드 목록에서 레이블을 확인할 수 있다.
```
$ kubectl get pods --show-labels

NAME              READY   STATUS    RESTARTS   AGE   LABELS
myapp-pod         1/1     Running   0          12m   <none>
myapp-pod-label   1/1     Running   0          14s   env=dev,tier=frontend
```

-L 옵션을 사용하여 특정 레이블을 지정하여 필드로 표시할 수 있다.
```
$ kubectl get pods -L env,tier

NAME              READY   STATUS    RESTARTS   AGE   ENV   TIER
myapp-pod         1/1     Running   0          13m
myapp-pod-label   1/1     Running   0          27s   dev   frontend
```

-o yaml 옵션에서도 레이블이 있다면 확인이 가능하다.
```
$ kubectl get pods myapp-pod-label -o yaml

apiVersion: v1
kind: Pod
metadata:
  creationTimestamp: "2021-03-02T02:39:12Z"
  labels:
    env: dev
    tier: frontend
...
```

### 5) 파드 레이블 수정
현재 존재하는 파드에 레이블을 추가하거나 이미 존재하는 레이블을 수정할 수도 있다.

앞서 생성한 레이블이 없는 myapp-pod 파드에 env 레이블을 추가해 보자.
```
$ kubectl label pods myapp-pod env=dev

pod/myapp-pod labeled
```

mynapp-pod-label 파드에 env 레이블을 수정해 보자.
```
$ kubectl label pods myapp-pod-label env=debug

error: 'env' already has a value (dev), and --overwrite is false
```

myapp-pod-label 파드에 dev 레이블은 이미 존재하며, debug로 수정하려고 했지만 이미 dev 값을 가지고 있다고 오류를 발생시켰다. 이미 레이블이 존재하는 경우 --overwrite 옵션을 사용하여 수정할 수 있다.

```
$ kubectl label pods myapp-pod-label env=debug --overwrite

pod/myapp-pod-label labeled
```

설정한 레이블을 확인해보자.
```
$ kubectl get pods -L env,tier

NAME              READY   STATUS    RESTARTS   AGE     ENV     TIER
myapp-pod         1/1     Running   0          15m     dev
myapp-pod-label   1/1     Running   0          2m54s   debug   frontend
```

### 6) 레이블 셀렉터
레이블 셀렉터는 오브젝트에 부여되어 있는 레이블을 식별하고 검색할 수 있다.

검색 방법은 두 가지가 있다.
- 특정 키가 있는/없는 레이블 포함
- 특정 키와 값이 있는/없는 레이블 포함

레이블 셀렉터를 이용하여는 검색하는 방법은 두 가지를 제공한다.
- 일치성 기준(Equality-based) 요건
  * =
  * ==
  * != 
- 집합성(Set-based) 기준 요건
  * in
  * notin 

### 7) 일치성 기준 레이블 셀렉터
tier 키가 포함되어 있는 레이블
```
$ kubectl get pods --show-labels -l tier

NAME              READY   STATUS    RESTARTS   AGE     LABELS
myapp-pod-label   1/1     Running   0          5m59s   env=debug,tier=frontend
```

tier 키를 제외한 레이블
```
$ kubectl get pods --show-labels -l '!tier'

NAME        READY   STATUS    RESTARTS   AGE   LABELS
myapp-pod   1/1     Running   0          18m   env=dev
```

env 키에 debug 값이 포함되어 있는 레이블
```
$ kubectl get pods --show-labels -l env=debug

NAME              READY   STATUS    RESTARTS   AGE     LABELS
myapp-pod-label   1/1     Running   0          6m30s   env=debug,tier=frontend
```

env 키를 가지지만 debug 값이 포함되지 않은 레이블 
```
$ kubectl get pods --show-labels -l 'env!=debug'

NAME        READY   STATUS    RESTARTS   AGE   LABELS
myapp-pod   1/1     Running   0          19m   env=dev
```

> 참고  
> ! 메타 문자가 쉘에 의해 해석되지 않게 따옴표로 묶어 준다.

### 8) 집합성 기반 레이블 셀렉터
dev 키에 debug 또는 dev 값이 포함되어 있는 레이블
```
$ kubectl get pods --show-labels -l 'env in (debug,dev)'

NAME              READY   STATUS    RESTARTS   AGE    LABELS
myapp-pod         1/1     Running   0          19m    env=dev
myapp-pod-label   1/1     Running   0          7m2s   env=debug,tier=frontend
```

tier 키를 가지지만 frontend 값이 포함되어 있지 않은 레이블
```
$ kubectl get pods --show-labels -l 'tier notin (frontend)'

NAME        READY   STATUS    RESTARTS   AGE   LABELS
myapp-pod   1/1     Running   0          19m   env=dev
```
