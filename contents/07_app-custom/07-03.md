## 3. 환경변수를 이용한 사용자화
시스템 내에서 쉘 환경변수는 env 명령으로 확인할 수 있다. 이전 서비스의 이름을 탐색하는 부분에서 환경변수를 확인해 본적이 있다.

도커에서는 아래와 같이 컨테이너에 환경 변수를 선언할 수 있다.
```
docker run --env NODE_PORT=8080 <image>
```
--env 옵션에 환경변수를 지정할 수 있으며, 일반적으로 환경변수의 변수명은 대문자로 한다.

### 1) 환경변수 테스트를 위한 이미지 빌드 및 생성
환경변수를 사용하도록 구성된 애플리케이션 이다.

> index.js 파일
```javascript
const http = require('http');
const os = require('os');
const ip = require('ip');
const dns = require("dns");

console.log(Date());
console.log("...Start My Node.js Application...");

var handler = function(request, response) {
    console.log(Date());
    console.log("Received Request From " + request.connection.remoteAddress);
    response.writeHead(200);
    response.write("Message: " + process.argv[2] + "\n");
    response.write("Hostname: " + os.hostname() + "\n");
    response.write("Platform: " + os.platform() + "\n");
    response.write("Uptime: " + os.uptime() + "\n");
    response.write("IP: " + ip.address() + "\n");
    response.write("DNS: " + dns.getServers() + "\n");
    response.end();
};

var www = http.createServer(handler);
www.listen(process.env.NODE_PORT);
```
이전에 사용했던 index.js를 조금 수정 한 버전이다. 이전에는 8080 포트를 지정하여 Node가 8080 포트로 서비스 하도록 구성하였는데, 이번에는 Node를 실행할 포트를 NODE_PORT 변수에서 참조하도록 구성된 애플리케이션이다.

> Dockerfile 파일
```dockerfile
FROM node:slim
WORKDIR /usr/src/app
COPY index.js .
RUN npm install ip
ENTRYPOINT ["node", "index.js"]
CMD ["Hello World!"]
EXPOSE 8080-8090/tcp
```
EXPOSE 노출 포트를 특정 포트가 아닌 범위로 지정하였다. 8080 포트부터 8090 포트를 사용할 수 있도록 하였다.

이미지를 빌드 하자.
```
docker build -t <ACCOUNT>/myweb:customport .
```
이미지의 이름은 myweb으로 똑같으며, 태그만 customport로 변경하였다.

도커 허브에 인증한다.
```
docker login
```

빌드된 이미지를 도커 허브로 푸시한다.
```
docker push <ACCOUNT>/myweb:customport
```

> 참고  
> c1t1d0s7/myweb-customport

### 2) 쿠버네티스 정의에서 환경변수 설정
쿠버네티스에서 환경변수를 정의 하려면 env 필드를 정의한다.

> mynapp-pod-env.yml 파일
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: mynapp-pod-env
spec:
  containers:
  - image: c1t1d0s7/myweb:customport
    name: mynapp
    env:
    - name: NODE_PORT
      value: "8088"
    ports:
    - containerPort: 8088
      protocol: TCP
```
이미지는 새로 빌드한 c1t1d0s7/myweb:custport를 사용하고, env 속성에 NODE_PORT 변수와 8088 값을 지정한다. .spec.containers.ports.containerPort 필드는 노출할 포트를 8088로 지정한다.

파드를 생성해 보자.
```
$ kubectl create -f mynapp-pod-env.yml

pod/mynapp-pod-env created
```

파드의 상세정보를 확인하면, Environment 필드에 설정된 환경변수를 확인할 수 있다.
```
$ kubectl describe pod mynapp-pod-env

...
    Environment:
      NODE_PORT:  8088
...
```

확인을 위해 포트포워딩을 구성한다. 포트는 8088에서 8088로 포워딩 한다.
```
$ kubectl port-forward mynapp-pod-env 8088:8088

Forwarding from 127.0.0.1:8088 -> 8088
Forwarding from [::1]:8088 -> 8088
Handling connection for 8088
```

새 터미널을 열어 확인해보자.
```
$ curl http://localhost:8088

Message: Hello World!
Hostname: mynapp-pod-env
Platform: linux
Uptime: 208216
IP: 10.233.76.41
DNS: 169.254.25.10
```
8088 포트로 서비스 하고 있음을 확인할 수 있다.

### 3) 리소스 삭제
```
$ kubectl delete -f mynapp-pod-env.yml

pod "mynapp-pod-env" deleted
```
