## 3. 환경변수를 이용한 사용자화
시스템 내에서 쉘 환경변수는 env 명령으로 확인할 수 있다. 이전 서비스의 이름을 탐색하는 부분에서 환경변수를 확인해 본 적이 있다.

도커에서는 아래와 같이 컨테이너에 환경 변수를 선언할 수 있다.
```
docker run --env MESSAGE="Hello Myweb" <IMAGE>
```
--env 옵션에 환경변수를 지정할 수 있으며, 일반적으로 환경변수의 변수명은 대문자로 한다.

### 1) 쿠버네티스 정의에서 환경변수 설정
쿠버네티스에서 환경변수를 정의 하려면 env 필드를 정의한다.

> myapp-pod-env.yaml
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp-pod-env
spec:
  containers:
  - image: ghcr.io/c1t1d0s7/go-myweb
    name: myapp
    env:
    - name: MESSAGE
      value: "Customized Hello World!"
    ports:
    - containerPort: 8080
      protocol: TCP
```

- .spec.containers.env: 환경 변수 정의
- .spec.containers.env.name: 환경 변수 이름
- .spec.containers.env.values: 환경 변수의 값

ghcr.io/c1t1d0s7/go-myweb 이미지는 MESSAGE 환경 변수가 있으면 해당 변수의 값을 출력 메시지로 사용하도록 만들어졌다.

파드를 생성해 보자.
```
$ kubectl create -f myapp-pod-env.yaml

pod/myapp-pod-env created
```

파드의 상세정보를 확인하면, Environment 필드에 설정된 환경변수를 확인할 수 있다.
```
$ kubectl describe pod myapp-pod-env

Name:         myapp-pod-env
...
Containers:
  myapp:
...
    Environment:
      MESSAGE:  Customized Hello World!
...
```

확인을 위해 포트포워딩을 구성한다. 포트는 호스트의 8080에서 컨테이너의 8080으로 포워딩 한다.
```
$ kubectl port-forward myapp-pod-env 8080:8080

Forwarding from 127.0.0.1:8080 -> 8080
Forwarding from [::1]:8080 -> 8080
```

새 터미널을 열어 확인해보자.
```
$ curl http://localhost:8080

Customized Hello World!
myapp-pod-env
```
변경된 메시지를 확인할 수 있다.

### 3) 리소스 삭제
```
$ kubectl delete -f myapp-pod-env.yaml

pod "myapp-pod-env" deleted
```
