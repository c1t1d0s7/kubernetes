## 4. 컨피그 맵
컨피그 맵(ConfigMap)

### 1) 컨피그 멥 정의 및 확인

#### (1) 컨피그 맵 생성 사용법
```
kubectl create configmap <NAME> [--from-file=[key=]source][--from-literal=key1=value1]
```

#### (2) --from-literal=key=value 형식 사용법
```
kubectl create configmap my-config1 --from-literal=key1=value1
```

#### (3) --from-file=source 형식 사용법
```
kubectl create configmap my-config2 --from-file=key2
```

#### (4) --from-file=key=source 형식 사용법
```
kubectl create configmap my-config3 --from-file=key3=key3.conf
```

#### (5) 컨피그 맵 확인
```
kubectl get configmap
```

```
kubectl get configmap my-config -o yaml
```

```
kubectl describe configmap my-config1
```

!!!추가!!!
#### (6) 오브젝트 파일을 이용한 컨피그 맵 정의
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: my-config4
data:
  key4: value4
```
!!!추가!!!

### 2) 컨피그 맵 생성 및 확인
```
mkdir configmap
echo "8080" > configmap/nodeport
echo "Hello Wolrd by ConfigMap" > configmap/message
```

```
configmap/
├── message [Hello World by ConfigMap]
└── nodeport [8080]
```

```
kubectl create configmap mynapp-pod-options --from-file=configmap
```

```
kubectl get configmap

```


### 3) 컨피그 맵을 참조하는 파드 생성
> mynapp-pod-cm.yml
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: mynapp-pod-cm
spec:
  containers:
  - image: c1t1d0s7/myweb:customport
    name: mynapp
    env:
    - name: NODE_PORT
      valueFrom:
        configMapKeyRef:
          name: mynapp-pod-options
          key: nodeport
    - name: MESSAGE
      valueFrom:
        configMapKeyRef:
          name: mynapp-pod-options
          key: message
    args:
    - $(MESSAGE)
    ports:
    - containerPort: 8080
      protocol: TCP
```

```
kubectl create -f mynapp-pod-cm.yml
```

```
kubectl port-forward mynapp-pod-cm 8080:8080
```

```
curl http://localhost:8080
```

### 4) 컨피그 맵의 볼륨 사용

#### (1) 컨피그 맵에 등록 할 설정파일
> nginx-gzip.conf
```
server {
    listen              80;
    server_name         mynapp.example.com;
    gzip on;
    gzip_types text/plain application/xml;
    location / {
        root   /usr/share/nginx/html;
        index  index.html;
    }
}
```

```
kubectl create configmap nginx-gzip-config --from-file=nginx-gzip.conf
```

```
kubectl describe configmap nginx-gzip-config
```

#### (2) 컨피그 맵을 볼륨으로 사용하는 파드 생성
> mynapp-pod-cm-compress.yml
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: mynapp-pod-cm-compress
spec:
  containers:
  - image: nginx
    name: mynapp-compress
    volumeMounts:
    - name: nginx-compress-config
      mountPath: /etc/nginx/conf.d
    ports:
    - containerPort: 80
      protocol: TCP
  volumes:
  - name: nginx-compress-config
    configMap:
      name: nginx-gzip-config
```

```
kubectl port-forward mynapp-pod-cm-compress 8080:80
```

```
curl -H "Accept-Encoding: gzip" -I http://localhost:8080
```

> 참고
subPath
```
spec:
  containers:
    volumeMounts:
    - name: nginx-config
      mountPath: /etc/nginx/conf.d/nginx-gzip.conf
      subPath: nginx-gzip.conf
```

```
kubectl describe pod mynapp-pod-cm-compress
```
