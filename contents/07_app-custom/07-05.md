### 1) 시크릿 정의 및 확인

#### (1) 시크릿 저장 데이터 종류
사용 가능한 시크릿 저장 데이터 종류
- generic: (기본) 키-값 형식의 임의데이터용 시크릿
- docker-registry: 도커 저장소 인증 정보용 시크릿
- tls: TLS 키 및 인증서용 시크릿

직접 사용가능하지 않은 시크릿 저장 데이터 종류
- service-account-token

표시되는 데이터 종류
- Opaque
- kubernetes.io/dockerconfigjson
- kubernetes.io/tls
- kubernetes.io/service-account-token

#### (2) 시크릿 생성 사용법
```
kubectl create secrets <TYPE> <NAME> [OPTIONS]
```
```
kubectl create secrets generic <NAME> [--from-file=[key=]source] [--from-literal=key1=value1]
```
```
kubectl create secrets docker-registry <NAME> --docker-username=user --docker-password=password --docker-email=email [--docker-server=string]
```
```
kubectl create secrets tls <NAME> --cert=cert-file --key=key-file
```

#### (3) Base64 인코딩 및 디코딩
```
echo "admin" | base64 > id.txt

YWRtaW4K
```
```
echo "P@ssw0rd" | base64 > pwd.txt

UEBzc3cwcmQK
```
```
cat id.txt | base64 --decode

admin
```
```
cat pwd.txt | base64 --decode

P@ssw0rd
```

#### (4) 시크릿 생성
```
kubectl create secrets generic my-secret --from-file=id.txt --from-file=pwd.txt
```

### 3) 시크릿을 이용한 HTTPS 웹 서비스 제공

#### (1) TLS 키 및 인증서 생성
```
openssl genrsa -out nginx-tls.key 2048
```

```
openssl req -new -x509 -key nginx-tls.key -out nginx-tls.cert -days 3650 -subj /CN=mynapp.example.com
```

!!!generic-->tls 확인!!!
#### (2) 시크릿 생성
```
kubectl create secret generic nginx-tls-secret --from-file=nginx-tls.key
--from-file=nginx-tls.cert
```

```
kubectl describe secrets nginx-tls-secret
```

#### (3) TLS 설정을 위한 Nginx 설정 파일 및 컨피그 맵 생성
> nginx-tls.conf
```
server {
    listen              80;
    listen		        443 ssl;
    server_name         mynapp.example.com;
    ssl_certificate	    /etc/nginx/ssl/nginx-tls.cert;
    ssl_certificate_key	/etc/nginx/ssl/nginx-tls.key;
    ssl_protocols	    TLSv1 TLSv1.1 TLSv1.2;
    ssl_ciphers		    HIGH:!aNULL:!MD5;
    location / {
        root   /usr/share/nginx/html;
        index  index.html;
    }
}
```

```
kubectl create configmap nginx-tls-config --from-file=nginx-tls.conf
```

```
kubectl describe configmap nginx-tls-config
```

#### (4) 시크릿 및 컴피그 맵을 사용하는 파드 생성
> mynapp-pod-https.yml
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: mynapp-pod-https
spec:
  containers:
  - image: nginx
    name: mynapp-https
    volumeMounts:
    - name: nginx-tls-config
      mountPath: /etc/nginx/conf.d
    - name: https-cert
      mountPath: /etc/nginx/ssl
      readOnly: true
    ports:
    - containerPort: 80
      protocol: TCP
    - containerPort: 443
      protocol: TCP
  volumes:
  - name: nginx-tls-config
    configMap:
      name: nginx-tls-config
  - name: https-cert
    secret:
      secretName: nginx-tls-secret
```

```
kubectl create -f mynapp-pod-https.yml
```

```
kubectl describe pod mynapp-pod-https
```

#### (5) HTTPS 웹 서비스 확인


### 5) 인그레스를 이용한 TLS 종료 프록시 구현

#### (1) TLS 종료 프록시 란?
TLS 종료 프록시(Termination Proxy)

#### (2) TLS 종료 프록시를 위한 TLS 인증서 생성
```
openssl genrsa -out ingress-tls.key 2048
```

```
openssl req -new -x509 ingress-tls.key -out ingress-tls.cert -days 3650 -subj /CN=mynapp.example.com
```

#### (3) TLS 키 및 인증서를 위한 시크릿 생성
```
kubectl create secret tls ingress-tls-secret --key=ingress-tls.key --cert=ingress-tls.cert
```

```
kubectl get secret
```

#### (4) TLS 종료 프록시를 위한 인그레스 컨트롤러 생성
> mynapp-ing-tls-termination.yml
```yaml
apiVersion: networking.k8s.io/v1beta1
kind: Ingress
metadata:
  name: mynapp-ing-tls-term
spec:
  tls:
  - hosts:
    - mynapp.example.com
    secretName: mynapp-tls-secret
  rules:
  - host: mynapp.example.com
    http:
      paths:
      - path: /
        backend:
          serviceName: mynapp-svc-ext-np
          servicePort: 80
```

```
kubectl create -f mynapp-ing-tls-termination.yml
```

#### (5) 인그레스 컨트롤러 확인
```
kubectl get ingresses.networking.k8s.io
```

```
curl --resolve mynapp.exampl.com:443:x.x.x.x -k -v https://mynapp.example.com
```
