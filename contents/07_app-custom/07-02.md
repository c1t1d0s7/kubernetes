## 2. 이미지 사용자화

### 1) 이미지 사용자화 정의
도커 이미지 빌드 시 사용하는 Dockerfile에는 이미지를 컨테이너로 실행했을 때 실행할 수 있는 애플리케이션을 정의하는 속성이 ENTRYPOINT와 CMD 두 가지 있다.

실제 이 두 가지 지시어를 어떻게 조합하는지에 따라 동작하는 방법이 다르다.

#### (1) ENTRYPOINT 만 사용하는 경우
- 실행 형식: ENTRYPOINT [“command”, “param1”, “param2”]
- 쉘 형싱: ENTRYPOINT command param1 param2

#### (2) CMD 만 사용하는 경우
- 실행 형식: CMD [“command”, “param1”, “param2”]
- 쉘 형식: CMD command param1 param2

#### (3) ENTRYPOINT와 CMD를 같이 사용하는 경우
- 실행 형식: ENTRYPOINT [“command”, “param1”] CMD ["param2]
- 쉘 형식: ENTRYPOINT command param1 CMD param2

ENTRYPOINT와 CMD를 같이 사용하는 경우는 ENTRYPOINT의 가장 첫 번째 항목은 명령어가 되며, CMD 항목은 무조건 파라미터가 되는 형식이다.

파드의 컨테이너 실행 시 CMD 항목은 다른 값으로 대체가 가능하다.

일반적으로 ENTRYPOINT와 CMD를 같이 사용하는 세 번째 경우를 가장 많이 쓰는데, 이는 실행할 명령어(애플리케이션)는 항상 똑같이 실행하고, 필요 시에 따라 파라미터를 변경하기 위함이다.

우리가 앞에서 생성한 이미지의 형태는 아래와 같다.
```dockerfile
ENTRYPOINT ["node", "index.js"]
CMD ["Hello World!"]
```
최종적으로 node index.js "Hello World!" 명령이 실행되는 형태이고, "Hello World!" 문장은 인수로 대체할 수 있다.

Node.js에 의해 실행되는 index.js는 node 명령의 두 번째 인수를 메시지 형태로 출력하도록 구성되어 있다.
```javascript
response.write("Message: " + process.argv[2] + "\n")
```

도커 명령에서 인스를 대체하려면 다음 명령을 사용한다.
```
docker run <image> <argument>
```
argument 부분에 원하는 문장을 지정하면, node index.js <arguement> 형태로 실행되게 되고, 이것이 index.js를 통해 메시지로 출력된다.

### 2) 쿠버네티스에서 인수 변경
Dockerfile에서 지정한 ENTRYPOINT 속성은, 쿠버네티스에서 컨테이너 정의시 command 속성으로 대체할 수 있고, CMD 속성은, 쿠버네티스에서 arg 속성으로 대체할 수 있다.

| 도커        | 쿠버네티스 |
|------------|---------|
| ENTRYPOINT | command |
| CMD        | args    |

다음은 인수를 변경하는 파드 오브젝트이다.

> mynapp-pod-arg.yml
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: mynapp-pod-arg
spec:
  containers:
  - name: mynapp
    image: c1t1d0s7/myweb
    args:
    - "Customized Hello World!"
    ports:
    - containerPort: 8080
      protocol: TCP
```

파드를 생성해보자.
```
$ kubectl create -f mynapp-pod-arg.yml

pod/mynapp-pod-arg created
```

메시지 변경을 확인하기 위해 포트포워드를 설정한다.
```
$ kubectl port-forward mynapp-pod-arg 8080:8080

Forwarding from 127.0.0.1:8080 -> 8080
Forwarding from [::1]:8080 -> 8080
Handling connection for 8080
```

새 터미널을 실행시켜 웹 컨텐츠를 확인한다.
```
$ curl http://localhost:8080

Message: Customized Hello World!
Hostname: mynapp-pod-arg
Platform: linux
Uptime: 207581
IP: 10.233.76.40
DNS: 169.254.25.10
```
변경된 메시지를 확인할 수 있다.

### 3) 리소스 삭제
```
$ kubectl delete -f mynapp-pod-arg.yml

pod "mynapp-pod-arg" deleted
```
