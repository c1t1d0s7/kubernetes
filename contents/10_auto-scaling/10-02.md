#### (2) 파드 수 계산

#### (3) 복제본 수 조정

### 2) HPA 생성 및 관리

#### (1) HPA를 위한 디플로이먼트
> mynapp-deploy-hpa.yml
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: mynapp-deploy-hpa
spec:
  replicas: 3
  selector:
    matchLabels:
      app: mynapp-deploy-hpa
  template:
    metadata:
      labels:
        app: mynapp-deploy-hpa
    spec:
      containers:
      - image: c1t1d0s7/myweb:v1
        name: mynapp
        resources:
          requests:
            cpu: 50m
            memory: 5Mi
          limits:
            cpu: 100m
            memory: 20Mi
        ports:
        - containerPort: 8080
```

```
kubectl create -f mynapp-deploy-hpa.yml
```

```
kubectl get deployments,pods
```

#### (2) HPA 생성
```
kubectl autoscale deployment mynapp-deploy-hpa --min 1 --max 5 --cpu-percent 70
```

```
apiVersion: autoscaling/v2beta2
kind: HorizontalPodAutoscaler
metadata:
  name: mynapp-hpa-cpu
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: mynapp-deploy-hpa
  minReplicas: 1
  maxReplicas: 5
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
```
- hpa.spec.metrics.type: 메트릭 소스 타입
    - Object
    - Pods
    - Resource
- hpa.spec.metrics.resource.target.type: 메트릭 타입
    - Utilization
        - hpa.spec.metrics.resource.target.averageUtilization 속성에 값 지정
    - Value
        - hpa.spec.metrics.resource.target.value 속성에 값 지정
    - AverageValue
        - hpa.spec.metrics.resource.target.averageValue 속성에 값 지정
```
