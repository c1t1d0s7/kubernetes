## 1. 파드의 리소스 관리

!!!수정!!! 1) 현재 리소스 사용량 확인 부분 아래 포함
### 1) 리소스 요청
리소스 요청(Requests)

#### (1) 현재 리소스 사용량 확인
```
kubectl top node XXXXX
```

```
kubectl top pod XXXXX
```

#### (2) 리소스 요청 정의
```yaml
containers: 
- resources:
    requests:
      cpu: 200m
      memory: 10Mi
```

#### (3) 리소스 요청 생성
> mynapp-pod-req.yml
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: mynapp-pod-req
spec:
  containers:
  - image: c1t1d0s7/myweb
    name: mynapp
    resources:
      requests:
        cpu: 200m
        memory: 10Mi
```

```
kubectl create -f mynapp-pod-req.yml
```

#### (4) 리소스 요청 확인
```
kubectl top pods mynapp-pod-req
```

```
kubectl describe nodes XXXXX
```

#### (5) 많은 리소스 요청을 요구하는 파드
> mynapp-pod-huge-req.yml
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: mynapp-pod-huge-req
spec:
  containers:
  - image: c1t1d0s7/myweb
    name: mynapp
    resources:
      requests:
        cpu: 4000m
        memory: 4Gi
```

```
kubectl create -f mynapp-pod-huge-req
```

```
kubectl get pods
```

```
kubectl describe pod mynapp-pod-huge-req
```

### 2) 리소스 제한
리소스 제한(Limits)

#### (1) 리소스 제한 정의
```yaml
containers: 
- resources:
    limits:
      cpu: 1
      memory: 20Mi
```

#### (2) 리소스 제한 생성
> mynapp-pod-limits.yml
```yaml

kind: Pod
metadata:
  name: mynapp-pod-lim
spec:
  containers:
  - image: c1t1d0s7/myweb
    name: mynapp
    resources:
      limits:
        cpu: 0.5
        memory: 20Mi

```
kubectl create -f mynapp-pod-limits.yml

#### (3) 리소스 제한 확인
```

```

```
kubectl top pods mynapp-pod-lim
```

```
kubectl exec mynapp-pod-lim -- md5sum /dev/zero
```

```
kubectl top pods mynapp-pod-lim
```

## 3) 제한범위 설정
제한범위(LimitRange)

### (1) 제한범위 생성
> mynapp-limitrange.yml
```yaml
apiVersion: v1
kind: LimitRange
metadata:
  name: mynapp-limitrange
spec:
  limits:
  - type: Pod
    min:
      cpu: 50m
      memory: 5Mi
    max:
      cpu: 1
      memory: 1Gi
  - type: Container
    defaultRequest:
      cpu: 100m
      memory: 10Mi
    default:
      cpu: 200m
      memory: 100Mi
    min:
      cpu: 50m
      memory: 5Mi
    max:
      cpu: 1
      memory: 1Gi
    maxLimitRequestRatio:
      cpu: 4
      memory: 10
  - type: PersistentVolumeClaim
    min:
      storage: 10Mi
    max:
      storage: 1Gi
```

```
kubectl create -f mynapp-limitrange.yml
```

#### (2) 제한범위 확인
```
kubectl get limitranges
```

```
kubectl describe limitranges mynapp-limitrange
```

### 4) 네임스페이스의 리소스쿼터
리소스쿼터(ResourceQuota)

#### (1) CPU 및 메모리 쿼터
> mynapp-quota-cpumem.yml
```yaml
apiVersion: v1
kind: ResourceQuota
metadata:
  name: mynapp-quota-cpumem
spec:
  hard:
    requests.cpu: 500m
    requests.memory: 200Mi
    limits.cpu: 1000m
    limits.memory: 1Gi
```

#### (2) 오브젝트 쿼터
> mynapp-quota-object.yml
```yaml
apiVersion: v1
kind: ResourceQuota
metadata:
  name: mynapp-quota-object
spec:
  hard:
    pods: 10
    replicationcontrollers: 2
    secrets: 10
    configmaps: 10
    persistentvolumeclaims: 5
    services: 5
    services.loadbalancers: 1
    services.nodeports: 2
    glusterfile.storageclass.storage.k8s.io/persistentvolumeclaims: 2
```

#### (3) 스토리지 쿼터
> mynapp-quota-storage.yml
```yaml
apiVersion: v1
kind: ResourceQuota
metadata:
  name: mynapp-quota-storage
spec:
  hard:
    requests.storage: 10Gi
    glusterfile.storageclass.storage.k8s.io/requests.storage: 5Gi
```

#### (4) 리소스 쿼터 생성 및 확인
```
kubectl create -f mynapp-quota-cpumem.yml -f mynapp-quota-object.yml -f mynapp-quota-storage.yml
```

```
kubectl get resourcequotas
```

```
kubectl describe resourcequotas mynapp-quota-cpumem
```

```
kubectl describe resourcequotas mynapp-quota-object
```

```
kubectl describe resourcequotas mynapp-quota-storage
```
