## 1. 노드네임
파드를 여러가지 이유나 관리 맟 효율적인 운영 목적상 특정 노드에 배치하기 위해 쿠버네티스의 파드 스케줄러를 제어하는 여러가지 방법에 대해 학습하자. 

사실 노드네임(nodeName)은 11장 파드 스케줄러"에서 유일하게 파드를 배치할 때 파드 스케줄러를 사용하지 않고, 파드 스케줄러보다 우선순위가 높은 방법이다.

노드네임은 파드를 특정 노드에 배치할 때 스케줄러가 관여하지 않고, 이후에 학슬할 노드 셀렉터, 어피니티, 테인트, 톨러레이션, 커든, 드레인은 파드 스케줄러를 제어하는 방법이다.
- 파드 스케줄러를 사용하지 않는 방법: 노드네임
- 파드 스케줄러를 사용하는 방법: 노드 셀렉터, 어피니티, 테인트, 톨러레이션, 커든, 드레인

파드를 특정 노드에 배치할 때 노드네임 기능은 그다지 추천하지 않는 방법이다. 그 이유는 다음과 같다.
- 지정된 노드가 존재하지 않으면 파드가 실행되지 않거나 컨트롤러에 의해 삭제될 수 있다.
- 지정된 노드의 리소스가 충분히 없는 경우, 파드가 실패함.
- 클라우드 환경에서 노드의 스케일링에 따라 노드 이름이 변하거나 고정되지 않는다.

노드네임 기능을 사용하기 위해 .spec.nodeName 필드를 사용하여 선언할 수 있다.

다음은 kube-node1 노드에 파드를 배치하기 위한 오브젝트다.

> mynapp-rs-nodename.yml
```yaml
apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: mynapp-rs-nn
spec:
  replicas: 2
  selector:
    matchLabels:
      app: mynapp-rs-nn
  template:
    metadata:
      labels:
        app: mynapp-rs-nn
    spec:
      nodeName: kube-node1
      containers:
      - name: mynapp
        image: c1t1d0s7/myweb
```

노드네임을 지정한 레플리카셋 리소스를 생성해보자.
```
$ kubectl create -f mynapp-rs-nodename.yml

replicaset.apps/mynapp-rs-nn created
```

파드가 어떤 노드에 배치됬는지 확인해보자.
```
$ kubectl get po -o wide

NAME                 READY   STATUS    RESTARTS   AGE   IP               NODE         ...
mynapp-rs-nn-57f6z   1/1     Running   0          11s   10.233.101.152   kube-node1   ...
mynapp-rs-nn-hdkbg   1/1     Running   0          11s   10.233.101.151   kube-node1   ...
```
역시 복제본 두 개 모두 kube-node1에 배치된 것을 확인할 수 있다.

레플리카셋 리소스의 복제본 개수를 세 개로 확장해 보자.
```
$ kubectl scale rs mynapp-rs-nn --replicas=3

replicaset.apps/mynapp-rs-nn scaled
```

추가된 파드가 어떤 노드에 배치됬는지 확인해보자.
```
$ kubectl get po -o wide

NAME                 READY   STATUS    RESTARTS   AGE    IP               NODE         ...
mynapp-rs-nn-57f6z   1/1     Running   0          111s   10.233.101.152   kube-node1   ...
mynapp-rs-nn-hdkbg   1/1     Running   0          111s   10.233.101.151   kube-node1   ...
mynapp-rs-nn-rnb9j   1/1     Running   0          30s    10.233.101.153   kube-node1   ...
```
역시 kube-node1 노드에 배치되는 것을 확인할 수 있다.

리소스를 삭제한다.
```
$ kubectl delete -f mynapp-rs-nodename.yml

replicaset.apps "mynapp-rs-nn" deleted
```
