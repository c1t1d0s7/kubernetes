## 2. 노드 셀렉터
노드 셀렉터(nodeSelector)는 파드를 특정 노드에 배치하기 위해 파드 스케줄러를 제어하는 가장 쉬운 방법이다.

노드 셀렉터는 노드에 레이블을 지정하고, 컨트롤러 및 파드 선언시 .spec.nodeSelector 필드로 레이블을 이용해 노드를 선택할 수 있다.

### 1) 노드 레이블 설정
만약 각 노드에 인공지능 애플리케이션을 위한 GPU 그래픽 카드가 있다고 가정하고 레이블을 선언해보자.

kube-node1 노드에 gpu 키와 highend 값을 가지는 레이블을 선언한다.
```
$ kubectl label nodes kube-node1 gpu=highend

node/kube-node1 labeled
```

kube-node2 노드에 gpu 키와 midrange 값을 가지는 레이블을 선언한다.
```
$ kubectl label nodes kube-node2 gpu=midrange

node/kube-node2 labeled
```

kube-node3 노드에 gpu 키와 lowend 값을 가지는 레이블을 선언한다.
```
$ kubectl label nodes kube-node3 gpu=lowend

node/kube-node3 labeled
```

노드에 선언된 리이블을 확인하자.
```
$ kubectl get nodes --show-labels

NAME           STATUS   ROLES    AGE   VERSION   LABELS
kube-master1   Ready    master   8d    v1.16.8   ...
kube-node1     Ready    <none>   8d    v1.16.8   ...gpu=highend...
kube-node2     Ready    <none>   8d    v1.16.8   ...gpu=midrange...
kube-node3     Ready    <none>   8d    v1.16.8   ...gpu=lowend...
```

### 2) 노드 셀렉터를 선언한 레플리카셋 생성
다음은 노드 셀렉터를 선언한 레플리카셋 리소스다.

> mynapp-rs-nodeselect.yml
```yaml
apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: mynapp-rs-ns
spec:
  replicas: 2
  selector:
    matchLabels:
      app: mynapp-rs-ns
  template:
    metadata:
      labels:
        app: mynapp-rs-ns
    spec:
      nodeSelector:
        gpu: highend
      containers:
      - name: mynapp
        image: c1t1d0s7/myweb
```
.spec.nodeSelector 필드에 gpu=highend 레이블을 지정하였다. 즉, kube-node1에 배치 될것으로 예상된다.

레플리카셋 리소스를 생성해보자.
```
$ kubectl create -f mynapp-rs-nodeselect.yml

replicaset.apps/mynapp-rs-ns created
```

### 3) 노드 셀렉터를 선언한 레플리카셋 확인
파드가 어떤 노드에 배치됬는지 확인해보자.
```
$ kubectl get po -o wide

NAME                 READY   STATUS    RESTARTS   AGE   IP               NODE         ...
mynapp-rs-ns-49smx   1/1     Running   0          40s   10.233.101.159   kube-node1   ...
mynapp-rs-ns-lb262   1/1     Running   0          40s   10.233.101.160   kube-node1   ...
```
역시 노드 셀렉터에 의해 kube-node1 노드에 배치되었다.

레플리카셋 리소스의 복제본 개수를 세 개로 확장해 보자.

```
$ kubectl scale rs mynapp-rs-ns --replicas=3

replicaset.apps/mynapp-rs-ns scaled
```

다시 파드가 어떤 노드에 배치됬는지 확인해보자.
```
$ kubectl get po -o wide

NAME                 READY   STATUS    RESTARTS   AGE   IP               NODE         ...
mynapp-rs-ns-49smx   1/1     Running   0          87s   10.233.101.159   kube-node1   ...
mynapp-rs-ns-7wgrs   1/1     Running   0          19s   10.233.101.161   kube-node1   ...
mynapp-rs-ns-lb262   1/1     Running   0          87s   10.233.101.160   kube-node1   ...
```
역시 kube-node1 노드에 배치되는 것을 확인할 수 있다.

### 4) 리소스 삭제
리소스를 삭제한다.
```
$ kubectl delete rs mynapp-rs-ns
replicaset.apps "mynapp-rs-ns" deleted
```
