## 4. 데몬셋

### 1) 데몬셋 소개
데몬셋(DaemonSet)은 노드 레이블과 매칭이 되는 모든 노드 또는 노드 레이블이 없다면 모든 노드에 하나씩의 파드를 동작시키는 컨트롤러이다. 

기능은 레플리케이션 컨트롤러와 레플리카셋과 비슷하지만, 복제본을 지정하지 않는다. 당연하게도 (노드 레이블과 매칭이 되는)모든 노드에 하나씩의 파드만을 배치하기 때문이다.

노드가 추가되면 자동으로 컨트롤러는 하나의 파드를 배치하게 되고, 노드가 제거되면 삭제된 파드를 다른 노드에 배치되지 않는다. 이는 복제본 컨트롤러가 아니기 때문이다.

### 2) 데몬셋 생성
데몬셋 오브젝트의 API 버전 역시 apps 그룹의 v1 버전을 사용한다.

> mynapp-ds.yml
```yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: mynapp-ds
spec:
  selector:
    matchLabels:
      app: mynapp-ds
  template:
    metadata:
      labels:
        app: mynapp-ds
    spec:
      nodeSelector:
        node: development
      containers:
      - name: mynapp
        image: c1t1d0s7/myweb
        ports:
        - containerPort: 8080
```
.spec.template.spec.nodeSelector 필드로 파드를 특정 노드에 배치할 수 있다.

> 참고  
> 노드의 레이블 셀렉터 필드는 레플리케이션 컨트롤러, 레플리카셋, 데몬셋, 잡, 디플로이먼트 컨트롤러에서도 사용할 수 있다.  

데몬셋 오브젝트를 생성해보자.
```
$ kubectl create -f mynapp-ds.yml

daemonset.apps/mynapp-ds created
```

### 3) 데몬셋 확인
생성한 데몬셋을 확인해보자
```
$ kubectl get daemonsets.apps

NAME        DESIRED   CURRENT   READY   UP-TO-DATE   AVAILABLE   NODE SELECTOR      AGE
mynapp-ds   0         0         0       0            0           node=development   5s
```

시간이 흐른 뒤 확인해보면 DESIRED, CURRENT, READY가 0 인 것을 알 수 있다. 당연하게도 파드는 생성되지 않는다.

오브젝트 정의에 노드셀렉터로 node=development 레이블을 선택하도록 정의하였다. 이 때문에 노드 레이블과 매칭되는 노드가 없기 때문에 아무 파드를 생성하지 않는다.

노드의 레이블을 확인하기 위해 --show-labels 옵션을 사용하면 노드에 설정된 레이블을 확인할 수 있다.
```
$ kubectl get nodes --show-labels

NAME           STATUS   ROLES    AGE   VERSION   LABELS
kube-master1   Ready    master   26h   v1.16.8   beta.kubernetes.io/arch=amd64...
kube-node1     Ready    <none>   26h   v1.16.8   beta.kubernetes.io/arch=amd64...
kube-node2     Ready    <none>   26h   v1.16.8   beta.kubernetes.io/arch=amd64...
kube-node3     Ready    <none>   26h   v1.16.8   beta.kubernetes.io/arch=amd64...
```

LABELS 필드에 보이는 값이 노드 레이블이다. 지금 확인되는 레이블들은 기본적으로 쿠버네티스 클러스터에 필요에 의해 사용하는 레이블이다. 해당 레이블에는 node라는 레이블의 키가 없으므로 매칭되는 노드가 없는 것이다.

### 4) 노드 레이블 지정
생성한 데몬셋 오브젝트가 파드를 생성하기 위해 노드에 레이블 추가로 지정해 보자. 추가할 레이블은 오브젝트에 정의한 node=development 레이블이다.

kube-node1 노드에 node=development 노드 레이블을 설정한다.
```
$ kubectl label nodes kube-node1 node=development

node/kube-node1 labeled
```

kube-node1 노드에 추가한 노드 레이블을 확인해 본다.
```
$ kubectl get nodes kube-node1 --show-labels

NAME         STATUS   ROLES    AGE   VERSION   LABELS
kube-node1   Ready    <none>   26h   v1.16.8   ...node=development
```

### 5) 데몬셋 확인
다시 데몬셋 상태를 확인해 본다.

```
$ kubectl get daemonsets.apps

NAME        DESIRED   CURRENT   READY   UP-TO-DATE   AVAILABLE   NODE SELECTOR      AGE
mynapp-ds   1         1         1       1            1           node=development   7m13s
```

파드가 실행된 것을 확인할 수 있다.

파드 목록도 확인해본다.
```
$ kubectl get pods

NAME              READY   STATUS    RESTARTS   AGE
mynapp-ds-nxvwq   1/1     Running   0          2m12s
```

### 6) 노드 레이블 제거
노드 레이블을 제거하여 데몬셋이 실행하는 파드를 제거할 수 있다. 레이블 제거는 레이블의 값을 제거할 수도 있고, 레이블의 키를 제거할 수도 있다. 키를 제거한다는 것은 값도 제거된다는 의미다.

#### (1) 노드 레이블 값 제거
```
kubectl label nodes kube-node1 node='' --overwrite
```
kube-node1 노드의 node 레이블의 값을 제거한다.

kube-node1의 노드 레이블을 확인해본다.
```
$ kubectl get nodes kube-node1 --show-labels

NAME         STATUS   ROLES    AGE   VERSION   LABELS
kube-node1   Ready    <none>   26h   v1.16.8   beta.kubernetes.io/arch=amd64,beta.kubernetes.io/os=linux,kubernetes.io/arch=amd64,kubernetes.io/hostname=kube-node1,kubernetes.io/os=linux,node=
```

#### (2) 노드 레이블 키 제거
노드 레이블 키 전체를 제거할 수 있다. 제거할 레이블 키 이름 뒤에 대시(-)를 사용한다.
```
$ kubectl label nodes kube-node1 node-

node/kube-node1 labeled
```

kube-node1의 노드 레이블을 확인해본다.
```
$ kubectl get nodes kube-node1 --show-labels

NAME         STATUS   ROLES    AGE   VERSION   LABELS
kube-node1   Ready    <none>   26h   v1.16.8   ...
```

둘 다 모든 경우 노드 레이블 셀렉터와 매칭이 되지 않기 때문에 파드는 제거된다.
```
$ kubectl get daemonsets.apps

NAME        DESIRED   CURRENT   READY   UP-TO-DATE   AVAILABLE   NODE SELECTOR      AGE
mynapp-ds   0         0         0       0            0           node=development   9m31s
```

### 7) 데몬셋 삭제
생성한 데몬셋 컨트롤러를 삭제한다.
```
$ kubectl delete -f mynapp-ds.yml

daemonset.apps "mynapp-ds" deleted
```
