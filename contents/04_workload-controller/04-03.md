## 3. 레플라카셋

### 1) 레플리카셋 소개
레플리카셋(ReplicaSet)

### 2) 레플리카셋과 레플리케이션 컨트롤러의 비교

### 3) 레플리카셋 생성
> 참고
```
kubectl api-resources
kubectl api-versions
```

> mynapp-rs.yml
```yaml
apiVersion: apps/v1beta2
kind: ReplicaSet
metadata:
  name: mynapp-rs
spec:
  replicas: 3
  selector:
    matchLabels:
      app: mynapp-rs
  template:
    metadata:
      labels:
        app: mynapp-rs
    spec:
      containers:
      - name: mynapp
        image: c1t1d0s7/myweb
        ports:
        - containerPort: 8080
```

> 참고
```
kubectl explain replicaset.spec.selector
```

### 5) 레플리카셋의 레이블 셀렉터 사용

#### (1) matchLabels 레이블 셀렉터
```yaml
spec:
  selector:
    matchLabels:
      key: value
```

#### (2) matchExpressions 레이블 셀렉터
```yaml
spec:
  selector:
    matchExpressions:
    - key: <string>
      operator: <In | NotIn | Exists | DoesNotExist>
      values:
      - <string>
```

### 6) matchExpressions를 사용한 레플리카셋 생성
> mynapp-rs-exp.yml
```yaml
apiVersion: apps/v1beta2
kind: ReplicaSet
metadata:
  name: mynapp-rs-exp
spec:
  replicas: 3
  selector:
    matchExpressions:
      - key: app
        operator: In
        values:
          - mynapp-rs-exp
      - key: env
        operator: Exists
  template:
    metadata:
      labels:
        app: mynapp-rs-exp
        env: dev
    spec:
      containers:
      - name: mynapp
        image: c1t1d0s7/myweb
        ports:
        - containerPort: 8080
```

```
kubectl create -f mynapp-rs-exp.yml
```
