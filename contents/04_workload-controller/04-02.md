## 2. 레플리케이션 컨트롤러

### 1) 레플리케이션 컨트롤러 소개
레플리케이션 컨트롤러(ReplicationController)는

### 2) 레플리케이션 컨트롤러 생성
> mynapp-rc.yml
```yaml
apiVersion: v1
kind: ReplicationController
metadata:
  name: mynapp-rc
spec:
  replicas: 3
  selector:
    app: mynapp-rc
  template:
    metadata:
      labels:
        app: mynapp-rc
    spec:
      containers:
      - name: mynapp
        image: c1t1d0s7/myweb
        ports:
        - containerPort: 8080
```

```
kubectl create -f mynapp-rc.yml
```

### 3) 레플리케이션 컨트롤러 확인
```
kubectl get replicationcontrollers
```

```
kubectl get pods
```

```
kubectl get pods --show-labels
```

```
kubectl get replicationcontrollers
```

```
kubectl describe replicationcontrollers mynapp-rc
```

### 4) 레플리케이션 컨트롤러와 파드
```
kubectl get pods
```

```
kubectl delete pods mynapp-rc-XXXXX
```

```
kubectl get pods
```

```
kubectl describe replicationcontrollers mynapp-rc
```

### 5) 레플리케이션 컨트롤러와 레이블

#### (1) 기존 파드에 레이블 추가
```
kubectl get pods --show-labels
```

```
kubectl label pods mynapp-rc-XXXXX env=dev
```

```
kubectl get pods --show-labels
```

#### (2) 기존 파드의 레이블 변경
```
kubectl label pods mynapp-rc-XXXXX app=test --overwrite
```

```
kubectl get pods --show-labels
```

```
kubectl label pods mynapp-rc-XXXX app=mynapp-rc --overwrite
```

```
kubectl get pods --show-labels
```

### 6) 파드의 수평 스케일링

#### (1) 명령을 이용한 스케일링
```
kubectl scale replicationcontroller mynapp-rc --replicas=4
```

```
kubectl get replicationcontrollers
```

```
kubectl get pods
```

#### (2) 오브젝트 수정을 이용한 스케일링
```
kubectl edit replicationcontrollers mynapp-rc
```
```yaml
...
spec:
  replicas: 5
...

replicationcontroller/mynapp-rc edited
```

```
kubectl get replicationcontrollers
```

```
kubectl get pods --show-labels
```

#### (3) 오브젝트 파일 수정을 이용한 스케일링
```
kubectl replace -f mynapp-rc.yml
```

```
kubectl get replicationcontrollers
```

#### (4) 레플리케이션 컨트롤러 및 파드 삭제
```
kubectl delete replicationcontrollers mynapp-rc
```

```
kubectl get replicationcontrollers
```

```
kubectl get pods
```
