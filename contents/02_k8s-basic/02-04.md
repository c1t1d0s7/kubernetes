## 4. Node.js 애플리케이션 컨테이너 이미지 만들기

> 참고  
> 본 과정에서 사용하는 도커 이미지는 이미 다 빌드되어 도커 허브에 업데이트 되어 있다.  
> 혹시 자신의 도커 허브 계정에 이미지를 푸시하고 싶은 사람만 진행하면 된다.  

### 1) Node.js 애플리케이션
간단한 Node.js 코드로 만들어진 웹 애플리케이션 이미지를 만들어보자.
> index.js
```javascript
const http = require('http');
const os = require('os');
const ip = require('ip');
const dns = require("dns");

console.log(Date());
console.log("...Start My Node.js Application...");

var handler = function(request, response) {
    console.log(Date());
    console.log("Received Request From " + request.connection.remoteAddress);
    response.writeHead(200);
    response.write("Message: " + process.argv[2] + "\n");
    response.write("Hostname: " + os.hostname() + "\n");
    response.write("Platform: " + os.platform() + "\n");
    response.write("Uptime: " + os.uptime() + "\n");
    response.write("IP: " + ip.address() + "\n");
    response.write("DNS: " + dns.getServers() + "\n");
    response.end();
};

var www = http.createServer(handler);
www.listen(8080);
```
위 Node.js 코드는 8080 포트로 애플리케이션에 접속 시 HTTP 응답코드 200을 리턴하며, HTTP 바디에 메시지, 호스트 이름, 플랫폼 종류, 업타임, IP, DNS 서버 주소가 리턴 되도록 하는 간단한 애플리케이션이다.

### 2) 사용자 지정 이미지 생성용 Dockerfile
위에서 작성한 index.js 파일을 포함하는 사용자 지정 이미지를 생성하기 위한 Dockerfile 이다.
> Dockerfile
```dockerfile
FROM node:slim
WORKDIR /usr/src/app
COPY index.js . 
RUN npm install ip
ENTRYPOINT ["node", "index.js"]
CMD ["Hello World!"]
EXPOSE 8080/tcp
```

- FROM
  * 이미지 생성 시 사용할 베이스 이미지 지정 
  * Node.js 기반의 애플리케이션을 실행하기 위해 Node 런타임이 기본 포함되어 있는 베이스 이미지 선택
- WORKDIR
  * 컨테이너 내에서 사용할 기본 작업 디렉토리를 지정
  * Node.js 이미지에서 웹 애플리케이션의 기본 디렉토리는 /usr/src/app
- COPY
  * 이미지 내에 추가로 복사할 파일의 소스와 타겟을 지정하는 부분이며 
  * 현재 디렉토리에 있는 index.js를 컨테이너의 기본 작업 디렉토리 인 /usr/src/app 디렉토리에 복사
- RUN
  * 사용자 지정 이미지 빌드 시에 실행할 명령(애플리케이션이 아님)이며 ip 라는 Node.js 패키지를 설치
- ENTRYPOINT
  * 이미지를 실행 했을 때 실행할 애플리케이션 지정
- CMD
  * ENTRYPOINT의 인수
- EXPOSE
  * 애플리케이션을 노출하기 위한 포트와 프로토콜 지정

즉, 최종적으로 이 이미지를 빌드한 뒤 이 이미지를 이용하여 컨테이너를 생성하면 기본 실행되는 애플리케이션(명령)은 ```node index.js ”Hello World!“ ```이다.

컨테이너 생성 시 실행할 애플리케이션을 지정하기 위해 ENTRYPOINT와 CMD를 사용할 수 있는데, 하나만 사용하는 방법이 있고, 둘 다 사용하는 방법 이 있다.

> 참고  
> https://docs.docker.com/engine/reference/builder/

### 4) 사용자 지정 이미지 테스트

#### (1) 컨테이너 실행
빌드 한 myweb:latest 이미지를 실행해서 잘 동작하는지 테스트 해보자.
```
$ sudo docker container run --name test -d -p 8080:8080 myweb

13877a227ed17bf7ee84d22e379a1d6bef40d4644f2ff81dcd55880c15df5492
```

호스트 포트를 8080 포트를 컨테이너의 8080 포트로 포워딩 하도록 구성한다.

#### (2) 컨테이너 확인
컨테이너의 상태를 확인해보자.
```
$ sudo docker container ls -l

CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS                    NAMES
13877a227ed1        myweb               "node index.js 'Hell…"   55 seconds ago      Up 54 seconds       0.0.0.0:8080->8080/tcp   test
```

#### (3) 애플리케이션 동작 확인
curl 명령을 이용해 http://localhost:8080 주소로 접속해보자.
```
$ curl http://localhost:8080

Message: Hello World!
Hostname: e19b9e70ba76
Platform: linux
Uptime: 11844
IP: 172.17.0.2
DNS: 10.233.0.3,127.0.0.53
```
위와같이 결과가 나온다면 이상없이 빌드된 이미지다. 혹시 이와같이 나오지 않는다면, 포트포워딩을 잘못했거나, 이미지 빌드시 제대로 빌드되지 않았거나, Node.js 애플리케이션이 문제이다.

#### (4) 컨테이너 종료
이미지를 삭제한다.
```
$ sudo docker rm -f test

test
```

### 5) 사용자 지정 이미지 업로드
사용자 지정 이미지를 도커 허브에 업로드 해보자, 우선 도커 허브에서 이미지를 다운받는 것은 별도의 절차가 필요 없지만, 업로드 하기 위해서는 도커 허브의 계정이 필요하다. 계정이 없다면 https://hub.docker.com에 접속해서 가입하자.

#### (1) 도커 허브 인증
도커 이미지를 업로드(푸시)하기 위해 인증이 필요하다.
```
$ sudo docker login

Username: XXXXX
Password: XXXXX

Login Succeeded
```
인증이 제대로 됬는지 확인하자.

#### (2) 사용자 지정 이미지 이름 변경
도커 허브에 이미지를 업로드 하기 위해서는 이미지 이름에 도커 허브의 계정 이름이 이미지에 있어야 한다.
기존에 빌드한 myweb:latest 이미지를 <계정>/myweb:latest 로 태그를 달자.
```
$ sudo docker image tag myweb:latest <ACCOUNT>/myweb:latest
```

이미지 목록을 확인해 이름이 이상없는지 확인하자.
```
$ sudo docker image ls

REPOSITORY      TAG     IMAGE ID        CREATED             SIZE
...
c1t1d0s7/myweb  latest  eee9c69a6f7f    15 minute ago       167MB
myweb           latest  eee9c69a6f7f    15 minute ago       167MB
...
```

#### (3) 사용자 지정 이미지 업로드
myweb 이미지를 푸시하자.
```
$ sudo docker image push <ACCOUNT>/myweb:latest
```

> 참고  
> 도커 허브에 업로드한 이미지는 별도의 조치를 취하지 않는 경우 기본적으로 공개 이미지로 업로드 된다.
