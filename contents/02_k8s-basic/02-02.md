## 2. YAML 및 오브젝트 기본
쿠버네티스 오브젝트는 쿠버네티스 시스템에서 정보를 담고있는 개체이다. 이런 오브젝트는 YAML 문법으로 정의된다.

### 1) YAML 이란?
YAML은 XML, C, Python, Perl, RFC2822(인터넷 메시지 포맷)에서 정의된 개념을 기반으로 만들어진 사람이 읽고 쓰기 쉽게 데이터를 정의하는 양식이다. 2001년 클라크 에반스에 의해 고안되었다.

YAML 이라는 이름은 YAML Ain’t Markup Language(YAML은 마크업 언어가 아니다) 라는 재귀적인 이름에서 유래 되었고, 원래 초기 YAML의 뜻은 Yet Another Markup Language(또 다른 마크업 언어)의 의미 였으나, YAML의 핵 심은 문서 마크업이 아닌 데이터 중심에 있다는 것을 보여주기 위해 이름을 바꾸었다.

최근에는 데이터의 직렬화를 위해 XML, JSON과 함께 YAML을 많이 사용한다.

### 2) YAML 요소
YAML은 모든 데이터를 스칼라, 리스트, 해시로 데이터를 표현한다.

#### (1) 스칼라/스트링
```yaml
banana
I am a boy
'I am a girl'
```
스칼라는 단순 간단한 문자 또는 문자열 이다. 보통 아무 표시를 하지 않아도 되나, 쌍/홑 따옴표를 사용하여 둘러쌀 수 있다.

#### (2) 리스트/어레이
```yaml
- Seoul
- Busan
- Incheon
```
하이픈으로 시작해여하며 한 줄에 하나의 요소를 표현한다.

#### (3) 해시/딕셔너리
```yaml
name: John Smith
age: 33
```
콜론을 이용하여 키:값 형태로 한 줄에 하나의 요소를 표현한다.

#### (4) 해시의 리스트
```yaml
- name: John Smith
  age: 33
- name: Mary Smith
  age: 2
```

#### (5) 리스트의 해시
```yaml
man:
  - John Smith
  - Bill Jones
women:
  - Mary Smith
  - Susan Williams
```

### 3) YAML 문법
- YAML의 문자열은 UTF-8 또는 UTF-16의 유니코드 문자집합을 사용함
- 공백 문자를 이용하여 들여쓰기로 계층구조를 구분함 (탭 문자는 들여쓰기에 사용하지 않음)
- 하나의 스트림에 있는 여러 개의 YAML 문서를 구분하기 위해 시작은 하이픈 3개(---), 끝은 마침표 3개(...)를 사용함
- 주석은 #으로 표시하며, 한 줄이 끝날 때까지 유효함
- YAML 파일의 확장자는 yml 또는 yaml을 일반적으로 사용한다.

### 4) 쿠버네티스 오브젝트
쿠버네티스는 클러스터의 상태를 나타내기 위해 오브젝트 개체를 정의하여 사용한다.

쿠버네티스에서 오브젝트를 생성할 때, 오브젝트에 대한 기본적인 정보와 함께 의도한 상태를 기술한 오브젝트 스펙(spec)을 제시해야 한다. 오브젝트를 생성하기 위해 우리는 YAML 또는 JSON 파일을 작성하며, kubectl 명령에 오브젝트가 정의된 YAML 또는 JSON 파일을 제공하며, kubectl은 JSON 형식으로 정보를 변환 시켜 API 요청을 한다.

다음은 파드 오브젝트를 기술한 간단한 YAML 파일의 예제 이다.

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myweb-pod
  labels:
    app: myweb
    type: frontend
spec:
  containers:
  - name: nginx-xontainer
    image: nginx
```

모든 오브젝트 정의 시 필수적으로 요구되는 필드
- apiVersion: 오브젝트를 생성하기 위한 API 버전
- kind: 오브젝트의 종류
  * 예: pod, services. replcaset, deployment 등
- metadata: name, UID, namespace 등을 포함하는 기본적인 정보
- spec: 오브젝트의 상태 정의
  * spec 내의 정의할 요소는 정의하고자 하는 오브젝트의 종류에 따라 다르다.

> 참고  
> 오브젝트 리소스 문서 확인 명령  
> 파드 오브젝트 리소스에 대한 예제  
> kubectl explain pod  
> kubectl explain pod.spec  
> kubectl explain pod.spec.containers  
> kubectl explain pod.spec.containers.name  
> kubectl explain pod.spec.containers.image  
> kubectl exaplin pod.spec --recursive

> 참고  
> vim 에디터로 YAML 파일 작성시 인덴트 및 하이라이팅, 들여쓰기 등 기능을 사용하여 효율적으로 작성할 수 있다.
> ~/.vimrc 파일에 아래와 같이 작성한다.
> ```syntax on```
> ```autocmd FileType yaml setlocal ts=2 sts=2 sw=2 expandtab autoindent```
