## 5. 애플리케이션 실행

### 1) 파드 생성
```
kubectl run myweb-1st-app --image=c1t1d0s7/myweb --port=8080 --generator=run/v1
```

> 참조: generator
https://kubernetes.io/docs/reference/kubectl/conventions/#generators

### 2) 파드 확인
```
kubectl get pods
```

### 3) 레플리케이션 컨트롤러 확인
```
kubectl get replicationcontrollers
```

### 4) 서비스 생성
```
kubectl expose replicationcontroller myweb-1st-app --type=LoadBalancer --name myweb-svc
```

### 5) 서비스 확인
```
kubectl get services
```

### 6) 서비스 접근
```
curl http://x.x.x.x:8080
```
또는
```
minikube service myweb-1st-app
```

### 7) 파드 스케일링
```
kubectl scale replicationcontroller myweb-1st-app --replicas=3
```

### 8) 스케일링 결과 확인
```
kubectl get pods
```

```
kubectl get replicationcontrollers
```

### 9) 서비스 요청 시 분산된 접근 확인
```
curl http://x.x.x.x:8080
```

```
curl http://x.x.x.x:8080
```

```
curl http://x.x.x.x:8080
```

### 10) 생성한 오브젝트 삭제
```
kubectl get all
```

```
kubectl delete pod myweb-1st-app-XXXXX
```

```
kubectl get all
```

```
kubectl delete replicationcontrollers myweb-1st-app
```

```
kubectl delete service myweb-svc
```

```
kubectl get all
```
