### 2) 서비스 생성

#### (1) 명령을 이용하는 방법
```
kubectl expose <CONTROLLER_TYPE> <CONTROLLER_NAME> [--type=<SVC_TYPE>] --name <SVC_NAME>
```

#### (2) YAML 파일을 이용하는 방법
> mynapp-svc.yml
```yaml
apiVersion: v1
kind: Service
metadata:
  name: mynapp-svc
spec:
  ports:
  - port: 80
    targetPort: 8080
  selector:
    app: mynapp-rs
```

```
kubectl create -f mynapp-svc.yml
```

### 6) 서비스의 세션 친화성 구성
> mynapp-svc-session-affinity.yml
```yaml
apiVersion: v1
kind: Service
metadata:
  name: mynapp-svc-ses-aff
spec:
  sessionAffinity: ClientIP
  ports:
  - port: 80
    targetPort: 8080
  selector:
    app: mynapp-rs
```

### 7) 서비스 다중포트 구성
> mynapp-svc-multiport.yml
```yaml
apiVersion: v1
kind: Service
metadata:
  name: mynapp-svc-mp
spec:
  ports:
  - name: mynapp-http
    port: 80
    targetPort: 8080
  - name: mynapp-https
    port: 443
    targetPort: 8443
  selector:
    app: mynapp-rs
```

### 8) 포트 이름 참조

#### (1) 포트 이름을 사용한 레플리카셋 및 서비스 생성
> mynapp-rs-named-port.yml
```yaml
apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: mynapp-rs-nport
spec:
  replicas: 3
  selector:
    matchLabels:
      app: mynapp-rs-nport
  template:
    metadata:
      labels:
        app: mynapp-rs-nport
    spec:
      containers:
      - name: mynapp
        image: c1t1d0s7/myweb
        ports:
        - name: mynapp-http
          containerPort: 8080
```

> mynapp-svc-named-port.yml
```yaml
apiVersion: v1
kind: Service
metadata:
  name: mynapp-svc-nport
spec:
  ports:
  - name: mynapp-http
    port: 80
    targetPort: mynapp-http
  selector:
    app: mynapp-rs-nport
```

```
kubectl create -f mynapp-rs-named-port.yml -f mynapp-svc-named-port.yml
```
