## 6. 헤드리스 서비스

### 1) 헤드리스 서비스 소개
헤드리스(Headless) 서비스

### 2) 헤드리스 서비스 생성
!!!추가!!!
> mynapp-rs-headless.yml
```yaml
apiVersion: apps/v1beta2
kind: ReplicaSet
metadata:
  name: mynapp-rs-headless
spec:
  replicas: 3
  selector:
    matchLabels:
      app: mynapp-rs-headless
  template:
    metadata:
      labels:
        app: mynapp-rs-headless
    spec:
      containers:
      - name: mynapp
        image: c1t1d0s7/myweb
        readinessProbe:
          exec:
            command:
              - ls
              - /var/ready
        ports:
        - containerPort: 8080
```

> mynapp-svc-headless.yml
```yaml
apiVersion: v1
kind: Service
metadata:
  name: mynapp-svc-headless
spec:
  clusterIP: None
  ports:
  - port: 80
    targetPort: 8080
  selector:
    app: mynapp-rs-headless
```

!!!변경!!!
```
kubectl create -f mynapp-rs-headless.yml -f mynapp-svc-headless.yml
```

### 3) 헤드리스 서비스 확인
```
kubectl get services
```

```
kubectl get endpoints mynapp-svc-headless
```

```
kubectl run nettool -it --image=praqma/network-multitool --generator=run-pod/v1 --rm=true bash

nslookup mynapp-svc-headless

nslookup mynapp-svc
```
