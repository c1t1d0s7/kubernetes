## 4. 인그레스

> 참고
```
kubectl get all -n kube-system -l app.kubernetes.io/name=nginx-ingress-controller
```

### 1) 인그레스 생성
인그레스(Ingress)

> mynapp-ing.yml
```yaml
apiVersion: extensions/v1beta1
kind: Ingress
metadata:
  name: mynapp-ing
spec:
  rules:
  - host: mynapp.example.com
    http:
      paths:
      - path: /
        backend:
          serviceName: mynapp-svc-ext-np
          servicePort: 80
```

```
kubectl create -f mynapp-ing.yml
```

### 2) 인그레스 컨트롤러 확인
```
kubectl get ingresses.extensions
```

```
curl --resolve mynapp.example.com:80:x.x.x.x http://mynapp.example.com
```

### 3) 다중 경로 인그레스

#### (1) 같은 FQDN의 다중 경로 인그레스 컨트롤러
- http://mynapp.example.com/web1
- http://mynapp.example.com/web2

> mynapp-ing-multi-paths.yml
```yaml
apiVersion: extensions/v1beta1
kind: Ingress
metadata:
  name: mynapp-ing-mpath
spec:
  rules:
  - host: mynapp.example.com
    http:
      paths:
        - path: /web1
          backend:
            serviceName: mynapp-svc-ext-np1
            servicePort: 80
        - path: /web2
          backend:
            serviceName: mynapp-svc-ext-np2
            servicePort: 80
```

```
kubectl create -f mynapp-ing-multi-paths.yml
```

```
kubectl get ingresses.extensions
```

```
kubectl describe ingresses.extensions mynapp-ing-mpath
```

#### (2) 다른 FQDN의 다중 호스트 인그레스 컨트롤러
- http://web1.example.com
- http://web2.example.com

> mynapp-ing-multi-hosts.yml
```yaml
apiVersion: extensions/v1beta1
kind: Ingress
metadata:
  name: mynapp-ing-mhost

  rules:
  - host: web1.example.com

  - host: web2.example.com
    http:
     paths:
      - path: /
        backend:
          serviceName: mynapp-svc-ext-np2

```

```
kubectl create -f mynapp-ing-multi-hosts.yml
```

```
kubectl get ingresses.extensions
```

```
kubectl describe ingresses.extensions mynapp-ing-mhost
```

### 4) 인그레스를 이용한 TLS 종료 프록시 구현

#### (1) TLS 종료 프록시 란?
TLS 종료 프록시(Termination Proxy)

#### (2) TLS 종료 프록시를 위한 TLS 인증서 생성
```
openssl genrsa -out ingress-tls.key 2048
```

```
openssl req -new -x509 ingress-tls.key -out ingress-tls.cert -days 3650 -subj /CN=mynapp.example.com
```

#### (3) TLS 키 및 인증서를 위한 시크릿 생성
```
kubectl create secret tls ingress-tls-secret --key=ingress-tls.key --cert=ingress-tls.cert
```

```
kubectl get secret
```

#### (4) TLS 종료 프록시를 위한 인그레스 컨트롤러 생성
> mynapp-ing-tls-termination.yml
```yaml
apiVersion: extensions/v1beta1
kind: Ingress
metadata:
  name: mynapp-ing-tls-term
spec:
  tls:
  - hosts:
    - mynapp.example.com
    secretName: mynapp-tls-secret
  rules:
  - host: mynapp.example.com
    http:
      paths:
      - path: /
        backend:
          serviceName: mynapp-svc-ext-np
          servicePort: 80
```

```
kubectl create -f mynapp-ing-tls-termination.yml
```
